# Редизайн UI: Разделение доменов (ZBB / Конверты)

Этот план описывает переход Laveh на строгую архитектуру разделенных контекстов, чтобы устранить смешивание задач на одном экране и реализовать логику конвертов.

## ⚠️ User Review Required
1. **Подход к планированию:** В YNAB будущие расходы планируются путем установки «Целей» (Targets) прямо внутри категорий бюджета (например, "Нужно накопить 50к к декабрю", и система сама просит отложить 10к в этом месяце). Мы можем пойти этим путем, **либо** сделать отдельный табличный экран «План» (матрица: месяцы x категории), чтобы видеть весь год вперед. Я предлагаю сделать **гибрид**: на экране «Бюджет» мы раскидываем деньги на текущий месяц, а на экране «План» видим горизонт на полгода вперед. Согласовано?
2. **Экран операций:** Оставляем ли мы отдельный таб «Операции» для ревью новых транзакций, или интегрируем ленту операций внутрь экрана «Счета»? (Рекомендую оставить отдельный таб, чтобы было удобно разгребать Inbox после импорта).

## Proposed Changes

### 1. Навигация и Маршрутизация
- Изменение состава вкладок: `Бюджет` (бывшая Главная), `Операции`, `Счета` (новый таб), `План`.

### 2. Экран «Бюджет» (Тактика)
- Убираем виджеты счетов и очередей транзакций.
- Главный фокус: блок `To Be Budgeted` (Свободные средства).
- Кнопка быстрого раскидывания денег («Ввести доход» / «Распределить»).
- Список категорий в формате ZBB. Для каждой категории выводим три колонки:
  - **Назначено (Assigned):** Сколько отложено в этом месяце.
  - **Активность (Activity):** Сколько потрачено в этом месяце.
  - **Доступно (Available):** `(Carryover + Assigned) - Activity`.
- Добавление флоу «Покрыть перерасход».

### 3. Экран «Счета» (Капитал)
- Выделение счетов в отдельный домен.
- Группировка: Наличные, Карты, Накопления, Долги.
- Главное действие: «Сверить баланс» (Reconcile). 

### 4. Экран «План» (Стратегия)
- Трансформация в "Матрицу целей/бюджетов" (горизонтальные месяцы, вертикальные категории).

### 5. Экран «Операции» (Inbox)
- Переносим `ReviewQueue` (непроверенные транзакции после импорта) на этот экран.
